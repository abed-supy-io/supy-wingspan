## supy-manifest — ArgoCD GitOps (Kubernetes manifests + Helm charts)

- **Purpose:** Source of truth for microservice deployments to GKE via ArgoCD; manages prod/dev/preview with image management + preview-env templating.
- **Structure:** `apps/` (~300 ArgoCD Application manifests, `supy-{service}-{env}/`; preview = `supy-{service}-{branch}/`); `charts/supy-frontend/` + external Helm charts (redis, unleash, postgres); `templates-preview/`; infra in `gke-monitoring/`, `keda-rbac/`, `pods-autoscaling/`.
- **Architecture & patterns:** App-of-Apps (`supy-app-of-apps.yaml`, auto-heal + prune); raw manifests + select Helm; images `europe-west4-docker.pkg.dev/.../{repo}/{branch}:{short-sha}`; Envoy sidecars as ConfigMaps; **Argo Rollouts blue-green** (settlements, manual promotion `autoPromotionEnabled:false`); **KEDA** preview auto-scale to 0; namespaces prod/dev/argocd/monitoring; node affinity `supy_dev_node`.
- **Tooling:** validate=**Datree** (CI on changed YAML) · CI=GitHub Actions (datree-validation.yml only) · lint=none · pre-commit=none · commits=conventional (image SHA tags; kyml templating) · deploy=ArgoCD (selfHeal). Scripts: create-app-manifest.sh, update-manifest.sh, delete-app.sh.
- **Testing:** Datree schema + misconfig checks on PR; no manifest unit tests.
- **Security / secrets / config:** K8s Secrets referenced (auth-tokens-keys, firebase-service-account-key-secret); ConfigMaps for common env; **known CI bug: hotfix image-tag corruption (first 9 chars replaced) — documented in CLAUDE.md, do not tamper**; no external-secrets/sealed-secrets; ArgoCD app-controller bound cluster-admin (KEDA requirement, documented trade-off); NetworkPolicy templates in postgresql-ha/rabbitscout.
- **Divergences vs typical k8s deploy practice:** blue-green with manual promotion (no canary); KEDA scale-to-0 for previews; heavy ConfigMap use; app-controller cluster-admin; hotfix tag-corruption managed by acceptance.
- **New patterns worth codifying:** kyml template vars ({{.APP_NAME}}/{{.IMAGE_TAG}}/{{.BRANCH_NAME}}/{{.APP_PATH}}) for preview gen; Datree CI policy rules (resource limits, securityContext, networkPolicy); App-of-Apps with `ignoreDifferences` (HPA-managed replicas); env promotion via directory naming (flat, no overlays).
- **Recommendation:** **policy-only + docs** — codify Datree policies (req/limits, securityContext, RBAC), document secret-management expectations (pairs with the supy-configmaps BLOCKING finding — deployments should consume `kind: Secret`), clarify blue-green-vs-Deployment policy. ArgoCD + Helm + raw manifests sufficient; no new tooling.
