## supy-scanner — Flutter federated plugin (native iOS/Android barcode+document scanner)

- **Purpose:** First-party drop-in Scanbot-compatible barcode/document scanner (AVFoundation+Vision / CameraX+ML Kit) replacing a paid third-party SDK.
- **Structure:** `lib/src/{channel,widgets,models,services,permissions,document,enhance}/`; `android/src/{main,test}/kotlin/io.supy.scanner/{barcode,document,perf,nativecore}/`; `ios/Classes/{barcode,document,invoice,nativecore,log,perf}/`.
- **Architecture & patterns:** Versioned MethodChannel (`io.supy.scanner/v1`) singleton boundary; sealed result types (no `dynamic` leaking); EventChannel-per-view for barcode stream; PlatformView (AndroidView/UiKitView) wrapper; controller pattern (pause/torch/formats); immutable value objects (`==`/`hashCode`/`toString`). Android CameraX + ML Kit with analyzer-thread isolation; iOS AVCapture on background DispatchQueue, VNRequest off main, results marshalled to main. **Not a BLoC app** — exposes ChangeNotifier + streams for consumers to wrap.
- **Tooling:** lint=flutter_lints + strict casts/inference/raw-types + forbidden patterns (no dynamic_calls, no runtimeType.toString, avoid catches without on-clause) · format=dart format (CI gate) · test=flutter_test + Robolectric (JVM) + XCTest; **coverage gate 70% lcov (Dart)** · CI=GitHub Actions (7 jobs: analyze+test, build Android/iOS, native tests JDK17, ios native tests, perfgate unit+emulator, enhance bench, canaries stable+JDK21) · codegen=none (hand-coded value types) · pre-commit=none documented · commits=conventional (scopes barcode/ios/android/document).
- **Testing:** flutter_test + Robolectric + XCTest; mocked MethodChannel for Dart units; formatMapper/documentScanner/deviceTier Kotlin tests; per-feature widget tests; **perfgate** device-tier perf benches with baseline JSON diffs; non-blocking canary jobs.
- **Security / secrets / config:** iOS 16+, Android minSdk 24 (NDK 26.1, C++17); versioned channel prevents silent breakage; GMS Document Scanner model pre-warmed; permissions via permission_handler wrapper; `debugForceTier` gated on `kDebugMode`; all on-device, zero network in scan path.
- **Divergences vs a typical Supy flutter repo:** library (not app) — no BLoC/go_router/get_it; native-heavy (C++ core v1.1, JNI/Swift bridge); versioned channel contract; perfgate harness; scanbot compat shim for drop-in cutover; sealed frozen UI-config types.
- **New patterns worth codifying:** (1) MethodChannel singleton wrapper w/ error translation + sealed results; (2) per-view EventChannel streaming; (3) native device-tier heuristic (cached, debug-overridable); (4) versioned channel name (`/v1`) convention; (5) immutable frozen config nested in sealed use-case enum; (6) hardware lifecycle binding isolated from widget lifecycle. **Native bridge hardening checklist (threading/cleanup/exceptions) is a distinct concern.**
- **Recommendation:** deepen flutter as a **plugin/package sub-profile** (distinct from app profile): codify MethodChannel singleton + sealed types, device-tier abstraction, per-view EventChannel, and a native-bridge hardening checklist. Perfgate is a candidate shared library.
