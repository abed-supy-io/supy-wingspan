## supy-mobile — Flutter monorepo (Clean Arch + BLoC + go_router + get_it + PageState sealed unions)

- **Purpose:** Data-driven inventory & procurement platform (retailer/supplier apps) — ordering, GRN, stock counting, requisitions, with fine-grained authz.
- **Structure:** Monorepo `apps/{retailer,connect,supy_storybook}` + 9 shared packages (core, design, authz_client, models, network, cache, firebase, sentry, api_query_tools). Each app feature: `<feature>/{application,data,domain,presentation,injection}`.
- **Architecture & patterns:** Clean Arch 3-layer (domain entities/repos/params; data datasources/models/repos; presentation BLoC/pages/widgets). BLoC mandatory (sealed events + freezed state). go_router with hardcoded paths in `lib/router/routes.dart`; sophisticated redirector chain (splash → force-update → auth → data-load → super-admin → authenticated). get_it `sl` singleton per feature. **Repositories return bare Futures (no dartz Either); exceptions via `throwAppException()` helper.** State wrapped in `PageState<T>` sealed union (init/loading/success/empty/failure) w/ freezed + pattern-match extensions. Dio + custom interceptors. RxDart BehaviorSubject.
- **Tooling:** lint=flutter_lints (VGA only in api_query_tools) · format=flutter format -l 120 · test=bloc_test + flutter_test + mocktail + integration_test · CI=GitHub Actions (retailer_unit_tests.yml — single workflow) · codegen=freezed, json_serializable, build_runner, gen-l10n · pre-commit=commitlint · commits=conventional.
- **Testing:** bloc_test + mocktail; integration_test (Firebase Test Lab); no explicit coverage bar; entry `test/unit_test.dart`.
- **Security / secrets / config:** flavors via flutter_flavor (default dev); `flutter_secure_storage` + `EncryptedStorageService` (in packages/core); `.env` via dotenv-vault per app; Sentry (supy_sentry); Firebase/Firestore; OTA via Shorebird; Unleash feature flags; Gleap in-app bug reporting; git-ignored `private/` for GCloud/asset keys.
- **Divergences vs a typical Supy flutter repo (vs checklist conventions):** (a) **no dartz Either — uses `PageState<T>` sealed union**; (b) repositories return bare Futures + `throwAppException()` (not Either); (c) VGA not at root (only api_query_tools); (d) bloc_lint not visible; (e) multi-stage redirector chain (more elaborate than checklist); (f) mixed sealed patterns (PageState @freezed, events `sealed`).
- **New patterns worth codifying:** (1) **Redirector Runner** — composable priority-chain route guards; (2) **`PageState<T>` sealed union over Either** (whenOrNull/maybeWhen + isLoading/isSuccess guards); (3) **`throwAppException()`** central exception translation; (4) reactive_forms ControlValueAccessor (LocationNameAccessor); (5) semantic `AppDialog` helpers over raw showDialog; (6) base BLoC inheritance (ChannelItemListBloc → DraftListEditBloc); (7) feature-scoped `injection/<feature>_dependencies.dart`.
- **Recommendation:** deepen flutter — but **the two Supy Flutter apps diverge on error handling** (checklist=dartz Either, supy-mobile=PageState+throwAppException). Standards must document BOTH as sanctioned or pick one. Push VGA + bloc_lint across all packages (currently only api_query_tools). Codify Redirector chain + PageState + design-package sharing.
