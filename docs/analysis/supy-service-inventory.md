## supy-service-inventory — NestJS 11 + Nx 21 + MongoDB (nestjs-nx) — canonical mined reference

- **Purpose:** Inventory domain service — real-time stock across locations, all stock movements (GRN receipts, wastage, transfers, production, stock counts).
- **Structure:** `apps/{api,migrate}` + `libs/{item,ledger,recipe,stock-count,transfer,wastage,productions,allowed-date,settings,common,big-query,context-maps}`.
- **Architecture & patterns:** strict CQRS + DDD; layer isolation api/ → logic/ → domain/ ← data/. Full CQRS (commands/+queries/) in stock-count, recipe, wastage, transfer, productions; partial in item (migration ongoing). Mongoose Input/Output Transformers; optimistic locking via `__v`. Domain events persisted polymorphically to `domainEvents` (fullName discriminator). `AggregateRoot` via `@supy.api/domain`; value-object IDs. Context-Maps facade (model→proxy→service) for inter-service NATS (catalog, grn, ledger, retailer, user). Exponential-backoff TransactionManager.
- **Tooling:** lint=ESLint 9 (FlatConfig, simple-import-sort, @nx/enforce-module-boundaries) · format=Prettier (@supy/prettier-config) · test=Jest 29 (co-located) · CI=no GitHub Actions (local-only) · codegen=Nx generators (library-group, library-feature-api/core/logic/data/queries/commands/contracts/mocks) · pre-commit=Husky (format + lint-fix) · commits=commitlint conventional.
- **Testing:** Jest 29, co-located specs, mock repos from `@supy/{lib}/mocks`, `Test.createTestingModule`; coverage bar undocumented.
- **Security / secrets / config:** `.env` (dotenv-cli) + `.env.vault` (Doppler); Firebase + GCloud Storage SA keys; NATS discovery via env; Mongo replica set; Redis (BullMQ); OpenTelemetry; `BOOTSTRAP_CAPABILITY` gating; dual transport (API NatsServer / Worker JetStream).
- **Divergences vs the plugin's documented nestjs-nx standard (stale/wrong?):** NestJS 11 import quirk (full path for `@Injectable()`); `nx.json` `sync.applyChanges:false` (non-standard, webpack-tsc workaround); partial CQRS in item is transitional; Context-Maps not yet universal. → standards should flag these as known issues.
- **New patterns worth codifying:** dual-transport runtime modes; bootstrap capabilities (enum-gated modules); polymorphic domain-event persistence; Input/OutputTransformer data layer; exponential-backoff TransactionManager.
- **Recommendation:** deepen nestjs-nx — flag NestJS 11 import quirk + sync.applyChanges workaround in docs; encourage Context-Maps adoption across services.
